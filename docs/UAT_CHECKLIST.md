# UAT Checklist - النظام الشامل لتوزيع حصيلة التنفيذ

## تعليمات تنفيذ UAT
- الهدف: التحقق من جاهزية النظام للاستخدام الفعلي داخل بيئة حكومية.
- البيئة: نسخة اختبار (Staging) مع بيانات تجريبية مماثلة للإنتاج.
- نطاق الاختبار: الأمن، الصلاحيات، الأداء الوظيفي، التقارير، تجربة المستخدم.
- نتيجة كل اختبار: Pass / Fail + ملاحظات + رقم التذكرة (إن وجد خلل).

---

## 1) المصادقة والأمن

### UAT-SEC-001: تسجيل دخول صحيح
- الخطوات:
  1. افتح شاشة الدخول.
  2. أدخل مستخدم وكلمة مرور صحيحة.
  3. اضغط دخول.
- المتوقع:
  - يتم الدخول للصفحة الرئيسية.
  - يتم تحميل بيانات المستخدم ودوره وإدارته.
  - لا تظهر أخطاء في الواجهة.
- الحالة: [ ] Pass [ ] Fail

### UAT-SEC-002: تسجيل دخول بكلمة مرور خاطئة
- الخطوات:
  1. أدخل اسم مستخدم صحيح وكلمة مرور خاطئة.
  2. كرر حتى 5 محاولات.
- المتوقع:
  - رفض الدخول في كل مرة.
  - بعد الحد المسموح يتم قفل الحساب مؤقتًا (axes).
  - يتم تسجيل محاولة الدخول الفاشلة في Audit Log.
- الحالة: [ ] Pass [ ] Fail

### UAT-SEC-003: تغيير كلمة السر
- الخطوات:
  1. سجل دخول.
  2. من الشريط العلوي اختر تغيير كلمة السر.
  3. أدخل الحالية + جديدة + تأكيد.
- المتوقع:
  - قبول العملية عند صحة المدخلات.
  - تسجيل الحدث في Audit Log.
  - إعادة تسجيل الدخول بكلمة المرور الجديدة.
- الحالة: [ ] Pass [ ] Fail

### UAT-SEC-004: صلاحيات RBAC حسب الدور
- الأدوار: admin / manager / officer / viewer.
- المتوقع:
  - viewer: عرض فقط، بدون إضافة/تعديل/حذف.
  - officer: إضافة/تعديل بدون حذف.
  - manager: إضافة/تعديل/حذف ضمن الإدارة.
  - admin: صلاحيات كاملة لكل الإدارات.
- الحالة: [ ] Pass [ ] Fail

### UAT-SEC-005: عزل البيانات حسب الإدارة
- الخطوات:
  1. سجل بمستخدم إدارة A.
  2. حاول الوصول لسجلات إدارة B.
- المتوقع:
  - منع عرض/تعديل/حذف بيانات خارج الإدارة للمستخدم غير admin.
- الحالة: [ ] Pass [ ] Fail

## 2) صفحة إدخال قسمة جديدة

### UAT-DIS-001: التحقق من قيود بيانات المدين
- المتوقع:
  - اسم المدين: حتى 40 حرف، لا يقبل أرقام.
  - الرقم المدني: 12 رقم فقط.
  - الحقول إلزامية.
- الحالة: [ ] Pass [ ] Fail

### UAT-DIS-002: التحقق من قيود بيانات القسمة
- المتوقع:
  - الرقم الآلي: 9 أرقام وينتهي بصفر.
  - نوع القسمة / نوع القائمة / التواريخ / الحصيلة: إلزامية.
  - الحصيلة تقبل حتى 3 منازل عشرية.
- الحالة: [ ] Pass [ ] Fail

### UAT-DIS-003: جدول الدائنين (AJAX + Inline)
- المتوقع:
  - إضافة دائن تعمل بدون إعادة تحميل الصفحة.
  - حذف دائن يعمل.
  - تعديل Inline يعمل.
  - قيود الرقم الآلي/المديونية تطبق فوريًا.
- الحالة: [ ] Pass [ ] Fail

### UAT-DIS-004: حساب القسمة القانونية
- سيناريو 1: الحصيلة تكفي المرتبة بالكامل.
- سيناريو 2: الحصيلة لا تكفي داخل مرتبة واحدة (قسمة نسبية).
- سيناريو 3: لا تنتقل المرتبة التالية عند نفاد الحصيلة.
- المتوقع:
  - الالتزام بترتيب المراتب المحدد.
  - نتائج دقيقة حتى 3 منازل عشرية.
- الحالة: [ ] Pass [ ] Fail

### UAT-DIS-005: الحفظ والطباعة
- المتوقع:
  - حفظ القسمة بنجاح.
  - توليد PDF للقسمة بعد الحفظ.
- الحالة: [ ] Pass [ ] Fail

## 4) التقارير والمستندات

### UAT-RPT-001: تقرير القسمة A4 متعدد الصفحات
- المتوقع:
  - Header رسمي + بيانات تعريفية + ترقيم صفحات (x من y).
  - توقيعات أسفل الصفحة.
  - تعدد الصفحات عند زيادة الدائنين.
- الحالة: [ ] Pass [ ] Fail

### UAT-RPT-002: تباليغ الحضور
- المتوقع:
  - اختيار القسمة عبر البحث بالرقم الآلي/المدني.
  - مستند صفحة مستقلة لكل شخص (المدين + الدائنين من الثاني للأخير).
  - يحتوي النص القانوني والتوقيعات.
- الحالة: [ ] Pass [ ] Fail

### UAT-RPT-003: محضر جلسة التوزيع
- المتوقع:
  - وثيقة صفحتين.
  - تحرير نص الصفحة الأولى/الثانية قبل التوليد.
  - Header رسمي + ترقيم صفحات + توقيعات.
- الحالة: [ ] Pass [ ] Fail
